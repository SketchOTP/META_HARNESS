## Objective

Implement full Slack integration for the Meta-Harness. After this directive:

- Every veto window posts to Slack with **Block Kit buttons** (Approve / Veto) so the operator can approve or abort a cycle from their phone without touching the terminal
- Cycle outcomes post a summary to Slack (COMPLETED ✓, TEST_FAILED ✗, VETOED ⊘, etc.)
- `/metaharness` slash commands work from any Slack client: `status`, `memory`, `pause`, `resume`, `proceed`, `veto`, `help`
- The daemon runs a **Socket Mode listener** in a background thread — no public URL required
- All Slack behavior is **optional** — if `[slack] enabled = false` or tokens are missing, the harness runs exactly as before with zero errors

---

## Architecture

```
slack_integration.py      — all Slack logic (Bolt app, handlers, formatters)
config.py                 — add SlackConfig dataclass + parse [slack] from toml
cycle.py                  — call slack hooks at veto open, veto result, cycle end
daemon.py                 — start Socket Mode thread before loop, stop on exit
cli.py                    — add `metaharness slack listen` and `metaharness slack test`
tests/test_slack.py       — unit tests (all Slack API calls mocked)
```

Slack is **never** imported at module load time in `cycle.py` or `daemon.py`. All hooks are called as optional function calls that check `cfg.slack.enabled` and return immediately if false.

#### Public functions

**`post_veto_window(cfg, directive_id, directive_title, directive_summary, seconds) -> Optional[str]`**

Posts a Block Kit message to `cfg.slack.channel`. Returns the Slack `ts` (message timestamp) which is used later to update the message. The message must contain:

- Header: `🔔 Veto Window Open — {directive_id}`
- Directive title as a section
- First 400 chars of directive summary (the diagnosis opportunities that led to this directive)
- Time remaining: `{seconds}s to approve or veto`
- Two buttons side by side:
  - `✅ Approve` — `action_id = "mh_approve"`, `style = "primary"`
  - `🚫 Veto` — `action_id = "mh_veto"`, `style = "danger"`
- Footer: `Delete .metaharness/PENDING_VETO to veto from terminal`

Block Kit structure:
```json
{
  "blocks": [
    {"type": "header", "text": {"type": "plain_text", "text": "🔔 Veto Window — {directive_id}"}},
    {"type": "section", "text": {"type": "mrkdwn", "text": "*{directive_title}*\n{summary}"}},
    {"type": "section", "text": {"type": "mrkdwn", "text": "⏱ {seconds}s veto window"}},
    {"type": "actions", "elements": [
      {"type": "button", "text": {"type": "plain_text", "text": "✅ Approve"}, "action_id": "mh_approve", "style": "primary"},
      {"type": "button", "text": {"type": "plain_text", "text": "🚫 Veto"},   "action_id": "mh_veto",   "style": "danger"}
    ]},
    {"type": "context", "elements": [{"type": "mrkdwn", "text": "Or delete `.metaharness/PENDING_VETO` to veto from terminal"}]}
  ]
}
```

Store the returned `ts` in `.metaharness/slack_veto_ts.txt` so the veto window poll loop can find it.

---

**`update_veto_result(cfg, approved: bool) -> None`**

Reads `ts` from `.metaharness/slack_veto_ts.txt`. Updates the message (using `chat.update`) to replace the buttons with a result line:
- Approved: `✅ Approved — proceeding`
- Vetoed: `🚫 Vetoed by operator`

Deletes `slack_veto_ts.txt` after updating.

```python
}
```

**`handle_slash_command(cfg, command: str, args: str) -> str`**

Handles `/metaharness <subcommand>`. Returns a string to respond with (Slack ephemeral response).

Subcommands:

| Command | Action |
|---------|--------|
| `status` | Read last 5 cycle JSONs from `cycles_dir`, format as table string |
| `memory` | Call `memory.compact_context(cfg, kg)`, return the compact string |
| `pause` | Create `cfg.daemon_pause_path`, return "Daemon paused ⏸" |
| `resume` | Delete `cfg.daemon_pause_path` if exists, return "Daemon resumed ▶" |
| `proceed` | Delete `cfg.pending_veto_path` — wait, this is wrong: to PROCEED, the veto file must be LEFT alone. Instead write `.metaharness/SLACK_EARLY_APPROVE` which `_veto_window` will detect as an early approval signal |
| `veto` | Delete `cfg.pending_veto_path`, return "Cycle vetoed ⊘" |
| `help` | Return help text listing all commands |

For `proceed`: the mechanism is a new file `.metaharness/SLACK_EARLY_APPROVE`. The `_veto_window` loop in `cycle.py` already polls for `PENDING_VETO` deletion. Add a second check: if `SLACK_EARLY_APPROVE` exists, delete it and return `True` immediately (proceed).

---

**`start_socket_mode(cfg) -> Optional[SocketModeHandler]`**

Creates a `slack_bolt.App` with the bot token, registers action handlers for `mh_approve` and `mh_veto`, registers a slash command handler for `/metaharness`, starts `SocketModeHandler` in a background thread. Returns the handler so the daemon can call `handler.close()` on shutdown.

Action handlers:
- `mh_approve`: calls `update_veto_result(cfg, approved=True)` and deletes nothing (veto file expires naturally — Approve means let it proceed)
- `mh_veto`: calls `update_veto_result(cfg, approved=False)` and deletes `cfg.pending_veto_path`

Slash command handler: parses body for command text, calls `handle_slash_command(cfg, ...)`, responds with `ack(text=result)`.

---

**`slack_test(cfg) -> None`**

Posts a test message: `"🤖 Meta-Harness Slack integration working — {project_name}"`. Used by `metaharness slack test` CLI command. Raises `RuntimeError` with a clear message if posting fails.

#### Token loading

```python
def _get_tokens(cfg) -> tuple[str, str]:
    """Return (bot_token, app_token). Raises RuntimeError if missing."""
    import os
    bot = os.environ.get(cfg.slack.bot_token_env, "")
    app = os.environ.get(cfg.slack.app_token_env, "")
    if not bot:
        raise RuntimeError(
            f"Slack bot token not found. "
            f"Set {cfg.slack.bot_token_env} environment variable."
        )
    if not app:
        raise RuntimeError(
            f"Slack app token not found. "
            f"Set {cfg.slack.app_token_env} environment variable."
        )
    return bot, app
```

## Acceptance Criteria

1. `pytest tests/` passes with zero new failures (67+ tests from D001 still pass, new Slack tests added).
2. `metaharness slack test` posts a visible message to the configured Slack channel.
3. `metaharness slack listen` starts without error and prints "Starting Slack Socket Mode".

8. `/metaharness pause` and `/metaharness resume` toggle the `DAEMON_PAUSE` file.
9. With `cfg.slack.enabled = False`, `metaharness run` and `metaharness daemon` work identically to pre-D003 behavior — no Slack imports, no errors.
10. With `slack-bolt` not installed, setting `cfg.slack.enabled = False` produces no import errors anywhere.

## Constraints

- Import `slack_bolt` and `slack_sdk` **only** inside functions that use them, guarded by `_SLACK_AVAILABLE`. Never at module top level in `cycle.py`, `daemon.py`, or `config.py`.
- The `run_cycle` public signature does not change.

- Socket Mode requires the app token starting with `xapp-`. Validate this prefix in `_get_tokens` and raise a clear error if wrong token type is provided.
